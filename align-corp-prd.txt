# Product Requirements Document (PRD)

## AtomQuest Hackathon 1.0

### In-House Goal Setting & Tracking Portal

---

# 1. Project Overview

## Project Name

**AtomGoal**
(you can rename later)

---

## Problem Statement

Organizations currently manage employee goals using fragmented systems like spreadsheets, emails, and manual review processes. This leads to:

* poor visibility
* weak accountability
* inconsistent tracking
* delayed reviews
* lack of centralized reporting

The objective is to build a web-based Goal Setting & Tracking Portal that digitizes:

* goal creation
* approvals
* quarterly tracking
* reporting
* governance workflows

---

# 2. Project Goal

Build a polished enterprise-style web portal where:

* Employees create and track goals
* Managers review and approve goals
* Admins oversee organization-wide progress

The system should support:

* structured workflows
* validation rules
* role-based access
* quarterly check-ins
* reporting dashboards
* audit logging

---

| Role         | Description                           |

# Validation Rules

## Rule 1

Total weightage = 100%

## Rule 2

Minimum goal weightage = 10%

## Rule 3

Maximum goals = 8

## Rule 4

No submission unless validations pass

# 12. Final Recommendation

Use:

* Google Stitch for UI generation
* Google AI Studio for demos/prompts
* Gemini CLI for rapid generation
* Supabase backend

Supabase is likely the better choice because:

* relational database fits this project
* role management is easier
* audit logs are cleaner
* dashboards become simpler

Firebase is faster for prototyping, but this project behaves more like ERP software than a realtime chat app.

Your biggest advantage now is:

> build fewer things, but make them feel production-ready 🏢✨
